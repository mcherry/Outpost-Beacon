Support

This is the official support page for Outpost Beacon. If you have a question, found a bug, or want to suggest a feature, you're in the right place.

Support Scope

Full support is provided for the Outpost Beacon app itself — the UI, built-in features, default check scripts, and the scripting API.

Custom check scripts are your own code. We can offer limited, best-effort guidance (pointing you to the right API function, explaining error messages, etc.), but we can't debug or troubleshoot custom scripts for you. The built-in editor's Run button and console output are the best tools for diagnosing script issues.

Common Questions

  • How do I add a custom service to monitor? — Settings → Scripts → click the + button to open the Script Wizard, or write a .js file and place it in the checks folder (Settings → Scripts → Open Scripts Folder).
  • How do I change how often services are polled? — Settings → General → Refresh Interval. You can also set per-script intervals with // OUTPOST_INTERVAL = "60" in the script header.
  • Why does a service show as "Unknown"? — The check script didn't call output(), or it threw an error. Open the script in the editor, click Run, and check the console output for errors.
  • Where is my data stored?
    • Scripts: ~/Library/Application Support/Outpost Beacon/checks/
    • Uptime database: ~/Library/Application Support/Outpost Beacon/uptime.sqlite
    • Preferences: macOS defaults system
  • How do I write a check script? — See the Scripting Guide, the included TEMPLATE.js.example in your scripts folder, or start with the Script Wizard which offers templates for Statuspage.io, HTTP health checks, TCP port checks, and more.
